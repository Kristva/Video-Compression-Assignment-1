In this implementation, I have elected to improve spatial modeling based on the adjacent left pixel and I have added 256 contexts. This is based on the idea that we reviewed in class, where we discussed how neighboring pixels within the same frame are usually similar to eachother, making it possible to estimate residuals based on the previously-loaded pixel. Instead of using one probability table for all pixels, I can then use 256 tables so the arithmetic coder builds separate probability estimates for different neighborhood types.

The compression rate achieved on the Bourne video using my method was 6.91. This is the highest outcome out of all of the clips I used. On the Roadrunner clip, the rate was 3.19, followed by 2.77 for The Social Network and 2.36 for Interstellar. I attempted to use different kinds of media to compare outcomes.

Interstellar likely had the worst compression ratio due to the fact that it contains a lot of granular color differences and textures in this specific scene. Roadrunner was the second best due to its color-blocked nature, where colors are grouped closely together but also have extremely different values in the places where transitions occur. The Social Network likely had a worse score also due to grain or noise.
